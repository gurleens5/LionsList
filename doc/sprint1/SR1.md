## Practices to Continue Next Sprint
- Writing clear branch names associated with tasks and creating pull requests for main.  
- Breaking user stories into small, manageable tasks and assigning them according to team expertise.  
- Reviewing pull requests before merging into main.  
- Adding and updating acceptance criteria for each user story on Trello.  

---

## New Practices to Try Next Sprint
- Add more detailed commit descriptions.  
- Increase the number of branches to align with tasks to keep pull requests organized.  
- Test features throughout development rather than waiting until the end of the sprint to catch bugs earlier.  

---

## Harmful Practices to Stop
- Updating Trello inconsistently, which caused confusion about task status.  
- Making multi-task commits, making PR reviews difficult and time-consuming.
