## Practices to Continue Next Sprint
- Re-evaluating and refining user stories as new features are added.  
- Start implementation of user stories well before the due date.  
- Write clear branch names associated with tasks and create pull requests for `main`.  
- Break user stories into small, clearly defined tasks and assign them to team members according to their expertise.  
- Add and update the acceptance criteria or tasks for each user story on Trello and in documentation.  

---

## New Practices to Try Next Sprint
- Prioritize testing and bug fixing over adding new features.  
- Thoroughly check edge cases and UI updates as features are added.  
- Conduct more frequent team check-ins to track completion of tasks/user stories and blockers.  

---

## Harmful Practices to Stop Next Sprint
- Avoid rushing changes near the end of the sprint.  
- Stop updating the Trello board inconsistently, which caused confusion about task status and made it harder to track progress.
