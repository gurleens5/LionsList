
# Definition of Done
# This Definition of Done has been agreed upon by all team members 

# Functionality

The feature works as expected from the corresponding user story
Functionality is fully complete
No feature causes incorrect or broken workflow in the system

# Integration

The feature integrates correctly with the other ones
Status changes happen properly when actions occur
No functionality is working incorrectly/broken

# Data Handling

The database stores all relevant information
Data updates properly after edits, deletions, or status changes
Data remains consistent after refreshing or restarting the application

# Correct Input Values

Validation occurs to make sure correct information is being entered
Invalid inputs display proper error messages
Only authorized users can perform specific tasks (e.g. only sellers can add listings)

# Testing

The feature has been tested thoroughly
Both valid and invalid inputs have been tested 
Edge cases have been tested against the feature
No critical or major bugs remain

# Code Quality & Review

Code has no errors 
The feature builds successfully
Changes have been reviewed by at least one team member
